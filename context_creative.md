## 통합 입력 페이지 (`creative_activity_input_v1_260507.html`)

### 진입 URL 규약
```
?type=자율활동|진로활동|동아리활동|봉사활동
&cls=2학년 1반
[&new=1]    # 새 학급 (선택)
[&demo=1]   # 데모 모드: 학생 자료 미리 채워짐 (기본은 empty)
```

### 페이지 구조 (전체 단계 공통)
```
[CLIPO 헤더 56px]
  ← 이전으로
  자율활동 : 2학년 1반                ← pt-title 22px/600
  학생 활동 자료나 특성 키워드를...      ← pt-desc 14px

[스텝 버튼 바]
  [1 학생 자료 입력] ▸ [2 콘텐츠 요소 선택] ▸ [3 기록 확인 및 수정]   [학생 추가]

[Step 1·2·3 패널 (한 번에 하나만 표시)]

[하단 sticky CTA]
  Step N/3 · {라벨}                      [← 이전] [다음 단계 →]
  진행률 텍스트
```

### Step 1: 학생 자료 입력 (✅ 완료)

- **카드 헤더**: `학생 자료 현황` + 진행률 sub + [+ 자료 등록] (primary)
- **테이블 컬럼**: 학번 / 이름 / 자료 수 / 키워드 / 성적 / 상태 / 관리
  - 모두 **중앙 정렬**, 본문 14px (CLIPO 컨벤션)
  - 학번은 5자리 (예: `20101` = 2학년 1반 1번)
  - 자료 수는 "N건", 키워드는 "처음 2개, 외 N", 성적은 상/중/하/—
  - 상태: 충분/부족/없음 (색 dot)
  - 관리: 충분 시 [수정] / 없음·부족 시 [+ 자료]
- **행 전체 클릭** → 학생 상세 모달
- **학생 상세 모달**:
  - 헤더: 이름 · 학번 · 활동 (1/8 nav 제거됨)
  - 활동 자료: dropzone(위) + 파일 리스트(아래, 신규는 아래 추가)
    - 파일 행: 클립 아이콘 + 이름 + (size 있으면) + ✕ 제거
  - 키워드: textarea (콤마 구분, 최대 500자, 글자수 카운터, auto-resize)
  - 성적 수준 칩: 상/중/하/고려 안함 (선택)
  - blur 시 자동 commit + refreshAll
  - [닫기] / [저장]

### [+ 자료 등록] 모달 (✅ 완료)

3개 시나리오 카드 가로 배치:
1. **학생 설문 응답 (EXCEL)** — 학번·이름이 포함된 설문 응답 결과 업로드
2. **활동지·보고서 (PDF)** — PDF 학급 스캔본 / 폴더 학생별 파일
3. **학생 관찰 키워드 (EXCEL)** — 양식을 다운받아 작성 후 업로드

카드 클릭 → 토스트 (해당 유형 상세 페이지는 미구현)

### 학생 추가 모달 (✅ 완료)

자율/진로/봉사: 학교 등록된 학생 중 학급에 추가 가능한 후보 (전학생 등) 리스트
- 헤더 sub: "학생이 보이지 않으면 [학교 설정 > 학생]에서 전학생을 먼저 추가해 주세요"
- 동아리 진입 시: 버튼 라벨 "학생 추가" → "학생 관리"로 전환
  - 학생 미등록(localStorage count=0): `dong_register_v4`
  - 학생 등록됨: `dong_student_manage_v1`

### Step 2: 콘텐츠 요소 선택 (✅ 완료)

**상단 헤더**: `콘텐츠 요소 선택 [AI 추천]` + 안내 + [전체 분석 시작 / 전체 재분석]

**2단 레이아웃**:
- 좌측 사이드바 (240px): 학생 목록 + 상태 뱃지
  - 자료없음 / 키워드만 / 미분석 / 분석중 / N/M (선택수/총항목)
- 우측 메인: 학생 헤더 + 상태별 카드
  - **자료 없음**: "1단계로 돌아가기"
  - **키워드만**: "AI 분석 불필요" 안내 + 키워드 요약
  - **미분석**: 자료 요약 + [AI 분석 시작]
  - **분석 중**: 스피너 + "약 30초 소요"
  - **완료**:
    - **종합 추천 근거** 카드 (overall_rationale + stats 4종)
    - **추천 항목** (체크 ON 기본) — content + 출처 뱃지 + ✦ 추천 근거
    - **추가 후보** (체크 OFF, 펼침/접힘 가능) — 옅은 톤
    - **직접 추가** — [+ 빈 카드 추가] → 노란 카드 + textarea
    - 학생 헤더에 [재분석] 버튼

**하단 CTA**: `분석 완료 N/M명 · 선택된 항목 X개`

**데이터 구조** (실제 API 응답 그대로):
```js
items: [{
  id, category,           // 산출물 기여 / 역할 수행 / 진로 연결 / 학기 성장
  content,
  sources: [{label, kind:'bulk'|'individual'|'keyword'|'auto'}],
  source_origin: 'stage1_recommended' | 'stage1_candidate',
  rationale,
  competency,             // 지식정보처리 / 창의적사고 / 자기관리
  cross_validated
}]
```

### 우측 하단 목업 토글 (개발용)

`MOCKUP [자료 1+ 있음] [키워드만]` — 시나리오 즉시 전환

- `STUDENTS_FULL`에 원본 보관, `applyScenario(scenario)` 함수로 전환

## 확정된 기획 결정

| 항목 | 결정 |
|------|------|
| **3단계 라벨** | 학생 자료 입력 → 콘텐츠 요소 선택 → 기록 확인 및 수정 |
| **용어 통일** | "데이터" 대신 **"자료"** (데이터 입력 → 자료 입력) |
| **분량/수준 입력 위치** | 분량은 step 3 생성 시점 / 수준은 학생 상세 모달 칩 |
| **키워드 입력** | textarea 콤마 구분 (태그 아님), 최대 500자 auto-resize |
| **자료 종류 (개별 업로드)** | DOCX · JPG · PNG · PDF (HWPX 다음 지원) |
| **일괄 업로드** | PDF (활동지) · XLSX (설문/관찰 키워드) |
| **분석 트리거** | 일괄 기본 + 학생별 재분석 가능 |
| **추천 vs 후보** | 별도 그룹 (추천 / 추가 후보) |
| **카테고리 표시** | API에 있으면 표시 (산출물 기여/역할 수행/진로 연결/학기 성장) |
| **직접 추가** | AI 카드와 동일 형태 빈 카드 인라인 편집 |
| **학번 형식** | 5자리 (학년+반+번호, 예 20101) |
| **본문 폰트** | 16px (CLAUDE.md 규칙). 단 테이블·라벨 등 informational text는 13~14px (CLIPO 컨벤션) |
