## 1. 과제물 관리 (OCR 확인·수정)

### 1-1. 비식별화 안내 (개인정보)
- **카피(고정)**:
  - 제목: `학생 개인정보는 안전하게 보호돼요`
  - 본문: `학번·이름은 자동으로 가려진 채 AI 채점돼요. 선생님 화면에는 그대로 보여서 채점에는 지장이 없어요.`
- **노출 방식**: 일괄 업로드(넓은 화면)=풀 안내 박스 / 개별 업로드 모달(좁음)=한 줄(`🔒 학번·이름은 가린 채 채점돼요`).
- **다시 보지 않기**: 풀 박스에 링크 → 작은 한 줄로 축소, `localStorage` 공통(일괄·개별·재방문 적용). 개별 한 줄은 상시(작아서 안 거슬림).
- 박스 스타일: 배경+아이콘만, **좌측 컬러 띠 금지**.
- 업로드 처리중 단계 표기: `개인정보 비식별화 중(학번·이름 가리는 중) → 글자 인식(OCR) → 분석 준비 완료`.

### 1-2. OCR 확인·수정 화면 (좌 원본 / 우 인식 결과)
- **학생 드롭다운(상단)**: `학번 이름` 형식(예 `10101 김서윤`) + 상태 아이콘 + 케이스 라벨. **OCR 안 된 학생(미진행·과제 미제출)은 비활성(선택 불가, 흐리게)**, 이전/다음 이동 대상에서도 제외.
- **좌측**: 과제물 원본 스캔(파일명·페이지·줌). 케이스에 맞는 원본 이미지.
- **우측 = OCR 인식 결과**, 문항별 카드 = **3섹션 구조**:

| # | 섹션 | 내용 | 수정 |
|---|---|---|---|
| ① | 문항정보 | 문항 번호 · 지시문 · (조건) | 읽기 |
| ② | 학생답변 | 인식된 답안 텍스트 | **수정 O** (클릭/연필 편집, 편집 시 `수식 입력` 버튼 노출) |
| ③ | 시각화자료 | 그림·그래프·도형 분석(Layer1 물리적 묘사 / Layer2 수학적 해석) | **수정 X**(`🔒 수정 불가` 뱃지, 회색 박스) |

- **채점·기타 흔적(빨간 동그라미·점수 마크)은 표시 안 함**(구조화 단계에서 제외).
- **그래프·그림만 답인 문항** → 학생답변 영역은 **빈칸 플레이스홀더**(`텍스트 답안 없음 — 시각화 자료를 참고해 채점해 주세요`), 시각화자료 섹션이 본체.
- **문항 라벨은 개조식 축약**으로 표기(예 `책에서 인상 깊었던 과학자 또는 원소의 발견 이야기를 소개하시오.` → `인상 깊었던 과학자 또는 원소의 발견 이야기 소개`). 단, **좌측 스캔 원본의 문제 글자는 원문 그대로**.
- **저장**: blur 자동저장 + **상단 `저장 후 다음` 주 버튼**(저장 토스트 + 다음 학생). 별도 하단 저장버튼·문항 수정 버튼은 숨김(문항 축약 표시라 수정 니즈 낮음).

### 1-4. 문항 단위 AI 채점 제외 (검토 중 — 줄리 제안 2026-06-25, AI팀 기술검토 요청됨)
- 배경: OCR 오인식 시 현재는 ① 교사가 답안 직접 수정 후 AI 채점 ② 수정 없이 그대로 AI 채점 — 둘뿐. 어느 쪽이든 30명×문항 답안·근거·피드백을 직접 손봐야 해 번거로움(특히 수식·그림 문항).
- 제안: OCR 단계에서 **문항마다 `이 문항 직접 채점 (AI 채점 제외)` 토글**. 체크 시 그 문항은 AI 채점 대상에서 빠지고 **교사 직접 채점**.
- 효과(채점 화면): 제외된 문항은 점수 직접 입력(AI 점수·근거·피드백 없음), **나머지 문항은 AI 채점**. 총점 = AI 문항 + 직접 문항 합산. → 학생 전체 직접 채점(인식 실패)보다 granular.
- 라벨: `직접 채점 (AI 제외)` 권장(줄리 스샷의 `교사 검수 필요`는 "확인하라"로 읽혀 모호 — 확정 필요).
- 상태: AI팀 기술검토 컨펌(2026-06-29). **HIAI 목업 반영 완료** — OCR 확인·수정 문항 카드에 토글, 채점 화면(좌측 문항 배지+딤, 우측 보라 안내 배너), 까망이 케이스(`문항 직접 채점`) 추가. 세션키 `hiai_ocr_exclude`로 화면 간 공유. CLIPO v1도 동일 반영.
- 구현 메모: 제외 상태 = `{ studentId: { 문항index: true } }`. OCR 토글 색=보라(직접 채점=교사 행위, 회색 비활성과 구분). 라벨 `직접 채점 (AI 제외)`. 채점 우측 패널은 채점기준 기반이라 문항 매핑 대신 보라 안내 배너로 반영(좌측 전사 패널은 문항 단위라 배지 직접 부착).

### 1-3. OCR 케이스 (드롭다운·셀에서 진입)
| 케이스 | 우측/안내 | 액션 |
|---|---|---|
| 정상(완료) | 인식 결과 정상 표시 | 확인·수정 |
| 확인 필요 | 일부 문항 불확실 → 해당 문항에 `AI 인식 불확실` 메모(어디가 왜) | 원본 대조 후 수정 |
| 정확도 낮음 | 불확실 ≥70% → 결과 미표시, `과제물 원본을 보고 직접 채점해 주세요` 안내 | 직접 채점 |
| 인식 실패 | `답안을 인식하지 못했어요`(글씨 흐림/스캔 불량) | **버튼 2개**: `다시 인식하기`(보조, 일시 오류용) + `파일 다시 올리기`(주, 품질 문제용). 재인식/재업로드 → 처리 후 **인식 성공 상태로 전환** |

## 2. AI 채점

### 2-1. 채점 실행 모달 (OCR 상태별 채점 가능/불가)
- 학생별로 채점 가능 여부 표기:
  - 채점 가능 → 체크 선택 + `AI 채점결과 N점`
  - 채점 불가 → **회색·비활성(선택 불가)**, 사유: `과제 미제출` / `인식 실패` / `정확도 낮음` (라벨에 `OCR ` 접두어 붙이지 말 것)
- 확인 필요(불확실 일부)는 **채점 가능**(선택됨). 헤더 카운트 `N/총명`.
- 점수 수준 슬라이더(더 낮은/기본/더 높은) — 기존 유지.

### 2-2. 학급 채점 상세 (명단 표)
- **명단 = 과제물 관리와 동일 로스터**(같은 학번·이름). 가짜/타학급 학생 끼우지 말 것.
- **OCR 상태 열 따로 두지 않음**(과제물 관리와 이중 정보). OCR 때문에 AI 채점 불가한 학생만 **채점 결과 칸에 사유 인라인**: `인식 실패 · 채점 불가 → 과제물 관리에서 확인` / `정확도 낮음 · 직접 채점 권장`.
- 정확도 낮음은 학급 단위에선 `확인 필요`로 묶어도 됨(집계 관점), 학생 상세에서만 분리.

### 2-3. 학생 채점 상세 — 케이스
| 케이스 | 우측 채점 영역 |
|---|---|
| 채점 진행(정상) | AI 채점 결과·근거·피드백 정상 + 선생님 직접 피드백 |
| 확인 필요 | 좌측 불확실 메모, 우측 채점 가능 |
| **OCR 미진행** | OCR 권장(주, 좌측 `과제물 관리에서 인식하기`) + **`직접 채점하기`(보조)** → 누르면 채점 UI 전환 |
| **인식 실패** | **직접 채점 화면**: 채점 결과(루브릭) 노출, **점수 미체크(총점 `- / 50`)**, **AI 채점 근거/피드백 섹션 숨김**, 선생님 직접 피드백은 유지. (AI 채점 불가, 교사 직접 채점만) |
| **재채점 필요(stale)** | **완료된 채점 화면(점수 채워짐) + 상단 주황 배너** |

핵심 구분: **인식 실패**=AI 불가→직접 채점이 유일(채점 UI 바로). **미진행**=OCR 돌리면 AI 가능→OCR 권장하되 직접 채점도 허용. **둘 다 채점 자체는 가능**(답안 인식 안 됐어도 교사가 점수 매김).

### 2-4. 재채점 유도 (stale) — 인식 바뀌면 채점 다시
- **트리거(실제 흐름)**: 과제물 관리에서 **OCR 수정 저장 / 다시 인식 / 파일 재업로드** 시 그 학생을 stale로 표시 → 채점 화면 진입 시 자동 노출. (HIAI 구현: `sessionStorage hiai_rescore` 배열에 학생ID 적재 → 학생 상세에서 감지. CLIPO도 동일 메커니즘.)
- **UI**: 학생 채점 상세 **상단 주황 배너**(주의 축) + **인라인 `재채점 실행` 버튼**. 학급 배너·확인 다이얼로그 없음.
  - 카피: `답안 인식 내용이 바뀌었어요. 다시 채점하면 최신 내용이 점수에 반영돼요.`
- **해소**: 재채점 실행 → "재실행 중→갱신됨" 토스트 + 배너 사라짐 + 세션 플래그 제거. 교사 확정값은 재채점 후 보존.
- 단일 플래그(학생 단위), 색은 **주황**(인식 실패=빨강과 구분).

### 2-5. 이동·이탈
- **`저장 후 다음`** 버튼(상단, 학생 상세·OCR 확인 공통).
- **`OCR 결과 수정`** = **새 탭 아니라 같은 페이지 이동**. 점수·피드백 등 **미저장 변경이 있으면 확인 모달**: `저장 안 하고 이동` / `저장하고 이동`(+취소). 변경 없으면 바로 이동.
- 새로고침 시 보던 화면 유지(세션 상태 저장·복원) — 캡처/리뷰 편의.

## 3. OCR 상태 사전 (의미·라벨·색 — 디자인 무관, CLIPO 토큰으로 치환)

| 의미 | 라벨 | 색 의미축 |
|---|---|---|
| 정상/완료 | 완료 | 🟢 정상 |
| 인식 중 | 진행 중 | 🔵 진행 |
| 검토 권장(고치면 채점 가능) | 확인 필요 | 🟠 주의 |
| 정확도 낮음(AI채점 막힘→직접) | 정확도 낮음 | 🔴 차단 |
| 인식 실패(결과 없음→재인식/재업로드) | 인식 실패 | 🔴 차단 |
| OCR 안 돌림 | 미진행 | 🟠 주의 |
| 교사 미확인 | 미확인 | ⚪ 대기 |
| 과제 미제출 | 미제출 | ⚪ 없음 |

- 색=의미 규칙: 🟢정상 · 🔵진행 · 🟠주의(액션하면 진행) · 🔴차단(AI채점 불가) · ⚪대기/없음.
- **같은 상태는 화면 불문 같은 라벨·색**(과제물 관리·채점·학생 상세 통일). CLIPO 파랑 계열 토큰으로 매핑하되 의미축 유지.
- 재채점 유도(stale)도 🟠주의(주황).

## 4. 문구 모음 (CLIPO도 동일 사용)
- 비식별화: `학생 개인정보는 안전하게 보호돼요` / `학번·이름은 자동으로 가려진 채 AI 채점돼요. 선생님 화면에는 그대로 보여서 채점에는 지장이 없어요.` (개별 한 줄: `학번·이름은 가린 채 채점돼요`)
- 인식 실패(우측): `답안을 인식하지 못했어요` / `글씨가 흐리거나 스캔 상태가 좋지 않아 답안을 읽지 못했어요. 다시 인식하거나 더 선명한 파일로 올려 보세요. 과제물 원본을 보고 직접 채점해도 됩니다.`
- 정확도 낮음: `과제물 원본을 보고 직접 채점해 주세요` / `확인이 필요한 문항이 많아(전체 N개 중 M개) OCR 인식 정확도가 낮아요. 인식 결과로는 AI 채점이 어려우니, 과제물 원본을 보고 직접 채점해 주세요.`
- 미진행: `OCR을 먼저 하면 AI 채점을 받을 수 있어요` / `답안 인식(OCR)을 끝내면 AI가 채점을 도와줘요. 왼쪽에서 과제물 관리로 이동해 인식하거나, 원본을 보고 바로 직접 채점할 수 있어요.`
- 재채점(stale): `답안 인식 내용이 바뀌었어요. 다시 채점하면 최신 내용이 점수에 반영돼요.`
- 직접채점 빈칸: `텍스트 답안 없음 — 시각화 자료를 참고해 채점해 주세요`
- 이탈 가드: `저장하지 않은 변경사항이 있어요` / `채점 내용을 저장하지 않고 OCR 결과 수정 화면으로 이동하면 변경사항이 사라져요. 저장하고 이동할까요?`
- 문항 라벨: 개조식 명사형 축약(예 `이 책의 핵심 내용 요약`, `과학 발전에서 국가 간 경쟁이 미친 영향 서술`, `그래프 개형 그리기`, `삼각형 넓이 구하기`).

## 5. 목업 내비게이터(까망이) — 시연용
- HIAI에 넣은 다크 플로팅 화면/케이스 선택기. CLIPO 목업에도 동일 포함(시연 편의, 제품 UI 아님). 정식명 "목업 내비게이터(Mockup Navigator)".
- 그룹: **화면**(채점 목록 등) → **AI 채점 실행 모달(OCR 상태별)** → **학생 채점 상세 케이스별**(채점 진행/확인 필요/OCR 미진행/OCR 인식 실패/재채점 필요/초등).
- 동작: 드래그·접기, 현재 화면 자동 활성표시, 파일 간 `?param` 점프, 케이스 리셋(반복 시연). 새로고침 시 보던 화면 유지.

## 6. CLIPO 포팅 체크리스트 (내일 작업 시)
1. CLIPO `task_ocr_inline_v1`(시안2) 또는 가장 가까운 OCR 파일 복제로 시작 — 헤더/NB/탭/푸터/토큰 유지.
2. OCR 확인·수정 우측을 **3섹션(문항정보/학생답변 수정O/시각화자료 수정X)**로 개편 + 문항 개조식 축약 + 학생답변 편집 시 수식 버튼.
3. OCR 케이스 4종(정상/확인 필요/정확도 낮음/인식 실패 2버튼) + 드롭다운 상태 아이콘·비활성(미진행/미제출).
4. 비식별화 안내(일괄 풀박스/개별 한 줄/다시 보지 않기) + 처리중 비식별화 스텝.
5. 채점 실행 모달 OCR 상태별 가능/불가(회색 비활성, 접두어 X).
6. 학급 채점 상세: 명단 동일 + OCR 불가 사유 채점 결과 칸 인라인(OCR 열 X).
7. 학생 채점 상세 케이스 5종(정상/확인필요/미진행=OCR권장+직접채점/인식실패=직접채점·점수미체크·AI기록숨김/재채점필요=완료+stale배너).
8. stale: OCR 수정·재인식·재업로드 → 세션 플래그 → 학생 상세 주황 배너+인라인 재채점 → 해소.
9. 저장 후 다음 / OCR 결과 수정 같은페이지 이동 + 미저장 이탈 가드 / 새로고침 화면 유지.
10. 까망이 목업 내비게이터(케이스 동일) 포함.
- CLIPO 차이 반영: 멀티파일(첫 파일만 채점)·직접입력 OCR 제외·6탭·표기 대문자 CLIPO.
