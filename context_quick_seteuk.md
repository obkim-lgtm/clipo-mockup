# 세특 간편 생성기 (간편 세특) 작업 컨텍스트

마지막 작업: 2026-05-26
다음 PC에서 이어받을 때 이 문서 먼저 읽기.

## 개요
교과 단위로 학생 세특 초안을 일괄 생성하는 흐름. 좌측 네비 `zap` 아이콘 활성.
배포: https://obkim-lgtm.github.io/clipo-mockup/output/ 아래

## 전체 흐름

```
[목록] ── + 교과 추가 모달 ──→ 카드 추가 (학생 등록 전 상태)
   │
   ├── 카드 클릭 (학생 0명) ──→ [참여 학생 등록]
   │                            저장 → ?registered=&count= → 목록 복귀 후 진행률 표시
   │
   ├── 카드 클릭 (학생 N명) ──→ [학생 자료 입력] (3-step 1단계)
   │                            → 2단계 AI 기록 생성 (미구현)
   │                            → 3단계 기록 편집 (미구현)
   │
   └── ⋮ 메뉴: 이름 변경 / 참여 학생 관리 / 삭제
```

## 파일

### 1. `output/quick_seteuk_list_v1_260526.html` — 목록
- 학년도별 3컬럼 카드 그리드
- 카드: `[과목] ⋮ / 제목 / 수정일 / 진행률 X/Y명 OR "학생 등록 전"`
- `card.dataset.classMode` = `'academic'|'mobile'` (반 편성 기준)
- `+ 교과 추가` 모달:
  - 기본 정보: 학년도(2026/2027) · 반 편성 기준 · 이름
  - 교과 및 성취기준: 교육과정 / 과목 / 성취기준 N개 (최대 10, 직접 입력 토글)
- ⋮ 메뉴 3개:
  - **이름 변경** → modal (브라우저 prompt 금지)
  - **참여 학생 관리** → 등록 페이지 이동
  - **삭제** → Figma red dialog 모달 (`아니오 / 네, 삭제합니다.`)
- 지난 학년도 카드는 진행률 회색 톤다운 (`:has(.ybadge.past)` 셀렉터)
- 우측 하단 MOCKUP 토글: `데이터 있음 / 데이터 없음` (드래그 가능)
- `?registered=&count=&subject=` 받으면 매칭 카드 진행률 갱신 또는 새 카드 prepend

### 2. `output/quick_seteuk_register_v1_260526.html` — 참여 학생 등록
- 헤더: `참여 학생 등록 : {그룹명}` + back-link (스텝바 없음 — 의도적)
- 2단 레이아웃 (좌: 학생 등록 방법 / 우: 콘텐츠)
- 등록 방식 3개:
  - 학생 목록에서 선택
  - 학생 명단 업로드 (엑셀)
  - 학생 명단+키워드 업로드 (엑셀 + 수행수준/키워드 컬럼)
- 학적기준반(`?classMode=academic`) vs 이동수업반(`?classMode=mobile`):
  - 학적: 학교 학생 트리에서 체크박스 선택
  - 이동: `+ 반 만들기` 모달 → 반 카드 아코디언 → 반별 학생 배정
- 반 카드 ⋮ 메뉴: 이름 변경 모달 / 삭제 모달 (Figma red dialog)
- 우측 하단 MOCKUP 토글: `학적반 / 이동수업반` (드래그)
- 저장 → `quick_seteuk_list?registered=&count=&subject=` redirect

### 3. `output/quick_seteuk_input_v1_260526.html` — 학생 자료 입력
- 3-step 헤더 (번호만, 아이콘 없음): `1 학생 자료 입력 / 2 AI 기록 생성 / 3 기록 편집`
- 좌측: 반 dropdown + 학생 목록(입력 chip · 자료 첨부 chip · 수행 chip) + **자료 일괄 등록 2개**(키워드 일괄 / PDF 일괄)
- 우측 (학생 선택 시):
  - **학생 활동 자료** 업로드 — PDF/JPG/PNG · **1개** · 최대 20MB
  - **키워드 사전** — 기본 접힘, 토글로 펼침 (강조 X)
  - 입력 textarea (키워드 태그 클릭 시 자동 추가)
  - **수행 수준 4개**: 상 · 중 · 하 · **자동** (PDF 있으면 자동, 없으면 중 디폴트)
- 검증: **자료 OR 키워드 중 하나** 입력 필수 (`저장 후 다음 학생` 막힘)
- 하단 sticky CTA: 진행률 + `이전 단계` (등록 페이지로) + `다음 단계` (미구현)

## 창체와의 관계
- `dong_register_v4` 구조 그대로 카피해서 세특 등록 페이지 시작
- 페이지 타이틀 톤 통일 (`동아리 참여 학생 등록 : XXX` ↔ `참여 학생 등록 : XXX`)
- "다른 양식은 사용할 수 없어요." 양쪽 동일
- 섹션 번호(`1.`, `2.`) 제거 — 양쪽 동일
- 다만 **세특 입력 페이지의 step bar는 안 만들기로 했음 (재논의 시점에)**
