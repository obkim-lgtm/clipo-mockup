# Clipo Mockup - 프로젝트 맥락

## 서비스 개요
교사의 서·논술형 평가 업무를 AI로 지원하는 B2B SaaS.
유사 해외 서비스: Pensive(pensive.com/schools), Co-Grader.
환경: PC 중심 웹 / React / OCR / AI 크레딧 과금 / 타겟: 중·고등 교사

## 핵심 플로우
회원가입·교사인증 → 수업 생성 → 평가 설계(성취기준+루브릭) → 답안 업로드 → AI 채점 → 결과 검토·수정 → 세특/행특 생성 → 저장

## 주요 화면
① 교사 홈
② 수업 관리
③ 평가 설계
④ 답안 업로드
⑤ AI 채점
⑥ 학생기록 생성 (세특/행특/창체)
⑦ 학교 관리
⑧ 이용권·크레딧
⑨ **랜딩 페이지(비로그인 홈)** — `output/landing_v2_260406.html`
⑩ **요금제 페이지** — `output/pricing_v1_260406.html`

## 랜딩 페이지 구성 (landing_v2_260406)
- 상단바: CLIPO 로고(클릭 시 랜딩) / 클리포 소개·보안 및 기술·이용 후기·가격(→요금제)·FAQ / 우측: 요금제(→요금제)·고객센터·가입하기·로그인
- 히어로: "채점은 쉽게, 기록은 빠르게 CLIPO AI 로 한 번에!"
- **체크리스트 배너** (히어로 직하): 다크 네이비 배경, 교육부 학습지원 SW 필수기준 체크리스트(공급자용) 다운로드
- 통계(5,000 도입학교 / 20,000 선생님 / 13.7M+ 실행)
- 클리포 소개 → 강점(스텝 탭) → 보안/기술 → 수상/리뷰 → 교사 후기 → 요금제 → FAQ → CTA → 푸터
- **사이드 배너 (fixed, right)**: 하나의 둥근 카드, TOP↑ + 소개서 + 에듀집 확인 + 도입 문의 — 아이콘 배경 없음, 항목 사이 구분선

## 요금제 페이지 구성 (pricing_v1_260406)
- 개인 / 학교 도입 탭
- 개인: 무료체험·60일·6개월·1년 4개 카드 (할인 뱃지)
- 학교 도입: 혜택 카드 + 수량 할인표 + 예상 견적 버튼
- 퀵링크 4개(소개서·에듀집·연수 혜택·연수 신청) → ※ 사이드 배너로 통합 검토 중
- 학운위 서류 다운로드 / AI 크레딧 추가 충전 / 학교 도입 문의 폼 / 결제 도움 3카드

## 핵심 설계 원칙
- 한 화면 = 한 가지 핵심 행동
- 교사 업무 시간 단축 우선, 예쁜 UI는 그 다음
- 다음 단계가 항상 명확하게 보일 것
- AI 결과는 항상 수정 가능 ("AI 초안 (수정 가능)" 표현 사용, 완성본처럼 보이지 않게)
- Primary 버튼 화면당 1개, 행동 중심 텍스트 (❌ 저장 → ✅ 채점 시작하기)

## 목업 작성 규칙
- 기본 형식: React JSX 컴포넌트 (Tailwind 또는 인라인 스타일)
- 컬러 토큰: primary #4A7FE8 기준, 기존 파일 스타일 그대로 이어받기
- 반드시 포함: 실제 데이터 예시, AI 결과 더미, 빈 상태(empty state), 버튼 클릭 후 흐름
- 기획 논의가 끝나면 바로 코드로 진입 (중간 설명 최소화)

## 벤치마킹 기준
- **Pensive** 참고 포인트: 루브릭 설계 UX, 채점 결과 뷰, 교사 피드백 플로우
- **Co-Grader** 참고 포인트: AI 채점 결과 표현 방식, 수정 인터페이스
- 두 서비스 UX 흐름 참고하되, 한국 교육 현장 언어와 생기부 작성 워크플로우에 맞게 조정

## 명단 모델 (수업 ↔ 학생 연결)

> ⚠️ 2026-04-29 기획 결정. 이전 스냅샷 모델은 폐기 예정.

### 핵심 원칙
**수업의 참여 학생은 학생 ID 스냅샷이 아니라 학급 멤버십에 대한 참조다.** 학교설정에서 학급 명단이 바뀌면 그 학급을 참조하는 모든 수업에 자동 반영된다.

### 반 편성 분기 (수업 만들기 시 라디오 선택)
| 반 편성 | 명단 모델 | UX |
|---------|----------|-----|
| **학적 기준반** (기본) | `학급(학년·반) 참조 + 추가 학생 + 제외 학생` | 학년·반 드롭다운만 고르면 끝. 학생 추가/제외 화면 스킵. 합반·청강 등 예외는 사후 "참여 학생 관리"에서 가감 |
| **이동수업반** | `이동반(roster) 1..N + 학생 개별 명단` | 현행 유지. 이동반 만들고 학생 등록(또는 엑셀 업로드). 1·2학기 재활용용 "다른 수업 이동반 복사" 기능 추가 예정 |

### 자동 동기화 규칙
- 학교설정에서 학급에 학생 추가 → 그 학급 참조하는 모든 수업에 자동 추가 (단, "제외" 목록에 없을 때)
- 학교설정에서 학생 정보 수정 → 모든 수업에 자동 반영
- 학교설정에서 학생을 다른 학급으로 이동 → 원 학급 참조 수업에서 빠지고, 새 학급 참조 수업에 추가
- 한 수업에서만 빼기 ("위탁생", "장기결석" 등) → 그 수업의 "제외 목록"에 기록. 학급 변동과 무관하게 유지

> ⚠️ **현재 미개발.** 위 규칙은 명단 모델의 미래 방향이며 아직 구현되지 않았다. **사용자 노출 카피(안내 박스 등)에서 "전학생이 들어오거나 빠지면 자동 반영" 같은 약속을 하지 말 것.** 모델 구현 후 카피 갱신.

### 마이그레이션
기존 수업의 학생 목록은 학급 참조로 자동 해석하지 않고, 새 수업부터 적용. (운영 부담 최소화)

## 인터랙션 안티 패턴 (절대 금지)

### ❌ 클릭 wrapper 안에 또 다른 클릭 요소 (button-in-button)
**라디오·버튼 등 클릭 가능한 wrapper(`<label>`, `<button>`) 안에 또 다른 클릭 요소(링크, 버튼, 셀렉트)를 넣지 않는다.** wrapper가 클릭 이벤트를 가로채서 내부 요소가 동작하지 않거나, 내부 요소를 누르면 의도치 않게 wrapper도 토글된다.

대표 사례:
- Radio Card 안에 "자세히 보기" 링크 → 링크 클릭 시 라디오만 토글되고 페이지 이동 X
- 버튼 안에 또 다른 버튼/아이콘 버튼 → 둘 중 어느 게 눌렸는지 모호

올바른 패턴:
- **라디오 + sibling 영역**: 라디오 라벨은 라디오만, 그 옆/아래에 별도의 영역으로 콘텐츠 구성 (이번 모달의 `radio-group` + `ban-sub`)
- **카드형 묶음이 꼭 필요하다면**: 카드 wrapper를 `<label>`이 아닌 `<div>`로 두고, 라디오 dot은 카드 내부의 작은 영역으로 한정. 이때도 카드 안 링크는 별도 클릭 핸들러로 명시.

### 원칙: 시각 정렬 < 기능 무결성
시각적 정렬·심미성 문제와 기능 동작 문제가 충돌할 때 **기능 우선**. 들여쓰기 정렬이 어색해 보여도 클릭이 막히는 것보단 낫다.
