## 확정된 정책·UI 결정

- **AI 크레딧 2분류**: 데일리 / 종량제. 옛 3분류(데일리/추가충전/보너스) 폐기. 각 카드에 성격 부연 한 줄(데일리=매일 충전·당일 소멸 / 종량제=잔량 유지).
- **종량제 유형 5종**: 추가 충전 · 학교 공유 · 보너스 · **온보딩** · 쿠폰(종량).
  - 온보딩 크레딧을 보너스에서 **분리** — 유효기간 **지급일+6개월**(당초 1개월로 오인→정정), 무료 사용자도 수령, 최대 여러 회.
  - 보너스 라벨은 결제수단(PG/기타) 무관 단일 라벨.
- **소진 순서**: 데일리 먼저 → 종량제. 각 트랙 안에서 **만료일 빠른 순**. 동률이면 쿠폰 → 학교 공유 → 보너스·온보딩 → 추가 충전(내돈내산 마지막). ← 옛 고정 9단계 대체.
- **종량제 내역 = 통합 모달 1개**로 전환(유형별 개별 상세 모달 폐기). 컬럼: 유형·발생일·출처사유·수량·잔여·만료일·상태. 만료일순/유형순 토글, 만료 건 회색·하단. 유형 태그는 무채색(상태 배지에 주목도 양보).
- **만료일 표기 룰**: 종량제 sub-row는 만료일 단일이면 인라인 표기, 여러 건이면 미표시(상세로). 날짜 포맷 `~YY.MM.DD`.
- **이용권 정보**: 이용 중/이용 예정 상태 배지. 선등록(이용 예정) 이용권을 화면에 노출(학교 예산 선결제·개인 순차결제 대응) — 시작일 표기, 시작일부터 데일리·보너스 적용.
- **개인 기타 결제(학교장터 등)**: 코드는 SCHOOL 트랙이나 UI엔 "개인 이용권"으로 표시.

- **상단바 2026 하반기 디자인** 반영: 메뉴 아이콘(panel-left-open), 요금제 배지, 크레딧 표기 "N 크레딧", 사용자 버튼(알약형). 이용권/AI 크레딧 **호버 팝오버** + "AI 크레딧 사용 안내" 모달. 팝오버는 만료일 빠른 순, 쿠폰명 미표시("쿠폰 크레딧" 일괄).
