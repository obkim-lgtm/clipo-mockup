# Principles
1. 팝업보다 인라인: 핵심 입력·선택·다음 단계는 페이지 인라인/전환(스텝)으로. 모달은 되돌릴 수 없는 짧은 확인이나 아주 짧은 보조 입력에만 쓴다.
2. 모든 클릭 요소는 버튼 콘텐츠 중앙정렬 5종 세트를 적용한다: inline-flex; align-items:center; justify-content:center; line-height:1; vertical-align:middle.
3. UI에는 시스템이 실제로 아는 것만 표시한다. AI/시스템은 성공/실패만 보여주고 신뢰도·추정원인·자동 품질판정(과검출/누락)은 금지(사람·QA 몫).
4. 정보 흐름: 조작 컨트롤(스코프/옵션)은 영향 필드 뒤에, 설명·잠금·경고 안내는 영향 필드 앞/상단에 둔다.
5. 본문 16px·레이블 14px 이상. Alert/Info 박스에 좌측 컬러 띠(border-left) 금지 — 배경색+아이콘만으로 구분.
6. 서비스명은 항상 대문자 CLIPO (소문자 Clipo/clipo 금지).
7. 새 목업은 새로 짜지 말고 개발 확정 화면(CLAUDE.md 개발 화면 기준 표)을 복제·확장한다 — 헤더·NB·탭·푸터·토큰 그대로.
8. 모달/CTA 버튼 레이블은 단어형(추가/저장/삭제), ~하기 금지. CSS 변경 후 ?v= 버전 쿼리를 갱신한다.
