### 1단계 모달
- **트리거**: 1단계 드로어 닫을 때 **활동지/설문 파일이 바뀐 학생** + **AI 기록 이미 보유** 시
- **타이틀**: "AI 자료 분석을 다시 할까요?"
- **본문**: "변경한 자료를 반영하려면 2단계에서 다시 분석해야 해요. 분석 후 3단계에서 기록까지 다시 생성하면 새 자료가 반영돼요."
- **버튼**: [나중에] / [지금 다시 분석] (→ 2단계로 이동 + 해당 학생 선택)

### AI 작업 중 차단 정책 (Jacob 제안 반영 / 2026-05-28 확정)

**작업 상태 2종** — 둘 다 **학생 단위**로 진행:
- **분석 중** — 2단계 AI 분석 진행 (`s.s2.status === 'analyzing'`)
- **생성 중** — 3단계 기록 생성 진행 (`s._generating === true`)
- 내부 헬퍼: `isStudentBusy(s)` = 분석 중 OR 생성 중 / `anyStudentBusy()` / `blockIfBusy(s)`

**차단 단위 (핵심)**
- **학생 단위 잠금**: 작업 중인 그 학생은 1·2·3단계 어디서든 자료/내용/수준/생성/다시 생성/가져오기 전부 비활성. **다른 학생은 계속 편집·작업 가능** (A 분석 중이어도 B는 자유).
- **일괄(전체) 버튼은 잠그지 않고 열어둠** (Jacob 제안 / 2026-05-28 확정): [일괄 자료 등록]·[전체 분석 시작]·[전체 기록 생성]·[전체 AI 기록 옮기기] 모두 클릭 가능.
  - 이유: 전체 분석·전체 기록 생성은 **선택 모달 내부에서 작업 중인 학생이 이미 선택 불가(태그 "작업 중"/"분석 중")로 제외**되므로 버튼을 잠글 필요 없음. 일괄 동작 자체가 작업 중 학생을 건드리지 않음.
  - [전체 AI 기록 옮기기]: 작업 중 학생은 일괄 처리 루프에서 자동 제외.
  - [일괄 자료 등록]: 진입 막지 않음. 작업 중 학생과의 자료 충돌은 **저장 시 서버 검증**으로 처리.
- 결과적으로 **한 학생은 분석↔생성을 동시에 못 하고 순차로만** 진행 (2단계 진행 중 3단계 시작 불가, 반대도 동일).

**차단 대상 버튼** (작업 중인 학생 단위로만 비활성)
| 단계 | 차단 대상 (그 학생만) |
|------|----------|
| 1단계 | 드로어 자료 추가·교체·삭제, 활동 내용 textarea, 참여 정도 칩 |
| 2단계 | (개별) 재분석, 추천 항목 선택·해제, 보충 항목 입력·저장 |
| 3단계 | (개별) 다시 생성·기록 생성·가져오기 |

> 일괄(전체) 버튼은 위 차단 대상이 아님 — 열어두고 선택 모달에서 작업 중 학생만 제외.

**UI 처리**
- disabled(흐리게 `opacity:.45` + `cursor:not-allowed`) + 통일 툴팁
- **통일 툴팁 문구**: `"AI 작업 중에는 수정할 수 없어요. 완료 후 다시 시도해 주세요"` (분석 중/생성 중 구분 없이 동일)
- 클릭 가드: 비활성 버튼이라도 핸들러에서 `blockIfBusy()` 로 토스트 처리 (이중 안전장치)
- **생성 중 행 표시**: 3단계 해당 행 AI 기록 셀에 스피너 + `"AI가 기록을 생성하고 있어요"` (분석 중은 `"자료를 분석하고 있어요"`)
- 진행 완료 후 자동으로 잠금 해제 (`refreshBusyControls()`)

**라우팅 강제 진입** (백엔드 영역, 목업 비범위)
- 일괄 등록 진입 버튼은 프론트에서 비활성. URL 직접 진입 등 강제 케이스는 별도 UI 처리 없이 **저장 시 서버 에러 메시지**로 처리.

→ race condition 방지 + 어떤 태그 보여줄지 모호한 케이스 제거

### 핵심 원칙
- **2단계 신호** (점·안내): **활동지/설문 파일이 바뀐 경우에만** 노출 (분석 결과가 stale이므로)
- **3단계 신호** (점·태그): **어떤 변경이든 저장되면** 노출 (기록이 stale이므로)
- **3단계 [다시 생성] 차단**: 자료(파일) 변경되었는데 재분석 안 한 경우 → 옛 분석 결과로 생성하면 안 됨 → 대신 **[2단계 자료 분석 →]** 링크 노출
- **재분석 완료 후**: data-side 변경(자료·내용)은 분석에 반영됨 → 3단계 액션 [다시 생성] 가능 (태그는 "분석 결과 갱신됨")

## 5. 1단계 드로어 닫기 후 흐름

```
드로어 닫기
   ↓
변경 사항 있음?
   ├─ NO → 평범하게 닫기
   └─ YES → 학생 stale 마킹 (3단계 점·태그 노출)
              ↓
        활동지/설문 파일 변경 AND AI 기록 보유?
           ├─ YES → 1단계 모달 "AI 자료 분석을 다시 할까요?" 노출
           │         ├─ [지금 다시 분석] → 2단계로 이동 + 해당 학생 자동 선택 (기존 분석 결과는 유지, 사용자가 재분석 선택)
           │         └─ [나중에] → 점·태그만 노출, 머무름
           └─ NO  → 모달 없음, 점·태그만 (활동 내용/활동 수준 변경 케이스)
```
