# CLIPO PRD 작성 가이드

두 가지 예시 PRD(H-2605-00 평가 홈 개선, H-2604-00 평가 설계, C-2604-01 개별채점 개선)를 기반으로 정리한 작성 지침.

---

## 1. 파일 ID 규칙

```
C-YYMM-NN  제목
```

- `C` : CLIPO (HIAI는 `H`)
- `YYMM` : 작성 연월 (예: 2604 = 2026년 4월)
- `NN` : 순번 (00, 01, 02…)
- 제목은 기능명 + 핵심 동작 한 줄 (예: 수행평가 개별 채점 개선)

---

## 2. 문서 구조

```
## 1. 도입 배경 및 목적
## 2. 화면 구성 개요          ← 신규 화면이면 포함, 기존 화면 수정이면 생략 가능
## 3. 핵심 정책               ← 개발자가 판단해야 할 분기·조건·정책만
## 4. 주요 범위 (Scope)       ← ✅ 포함 / ❌ 제외
## 5. 유저 플로우             ← 코드블록 트리
## 6. 후속 과업               ← 있을 때만
## 목업
```

---

## 3. 섹션별 작성 원칙

### 1. 도입 배경 및 목적
- **왜 지금인가** + **무엇이 달라지는가** 2가지만
- 기획 의도(사용자 가치) → 개발 목표 순서로
- 후속 과업과 연결되는 경우 마지막 줄에 명시

> 예시
> - AI 채점 결과를 교사가 검토하고 확정하는 과정에서, 채점요소별 피드백이 별도 영역에 분리되어 있어 활용이 낮고 편집 경험이 불편했다.
> - 채점기준별 근거·피드백을 해당 요소 카드 내 인라인으로 통합하여 **채점과 피드백 작성을 한 흐름으로 처리**

### 2. 화면 구성 개요
- 탭 구조, 레이아웃 분할, 진입 경로가 있을 때만 작성
- 표 형태로 간결하게

### 3. 핵심 정책
**포함할 것**
- 상태 분기 조건 (어떤 상태에서 어떻게 동작하는가)
- 데이터 처리 규칙 (저장·삭제·덮어쓰기 정책)
- 단계 진입 조건 / 이탈 처리
- 정렬·노출 기준 (기본값 포함)

**포함하지 말 것**
- 색상·폰트·간격 등 UI 스타일 (목업으로 전달)
- 이미 반영된 기존 정책 반복
- "~할 수 있다" 식 기능 나열 → 조건과 결과로 바꿔 쓸 것

**변경 PRD는 Before → After 표로**

| 항목 | 기존 | 변경 |
|------|------|------|
| 과제물 탭 | 파일 목록 탭 UI | 드롭다운으로 교체 |

### 4. 주요 범위 (Scope)
- ✅ 포함: 이번 배포에 반드시 들어가는 것
- ❌ 제외: 논의됐지만 뺀 것 + 이유 한 줄 (이유 없으면 생략)

> 제외 이유 예시: `반(학급) 단위 진행 현황 분리 (복잡도 이슈로 전체 집계만 노출)`

### 5. 유저 플로우
- 코드블록 트리 형식 (들여쓰기로 분기 표현)
- 화면 이름은 대괄호 `[화면명]`
- 분기점은 `└─`, `├─`

```
[시작 화면]
  └─ 액션
        ↓
[다음 화면]
  ├─ 분기 A → 결과 A
  └─ 분기 B → 결과 B
```

### 6. 후속 과업
- 이 PRD가 선행 조건이 되는 다음 과업만 기재
- 표 형태: 과업명 | 내용

### 목업
- GitHub Pages 배포 URL 링크 1줄
- 로컬 URL(localhost) 사용 금지

---

## 4. 분량 기준

| 유형 | 적정 분량 |
|------|-----------|
| 신규 기능 | 핵심 정책 3~5개 섹션, A4 2장 이내 |
| 기존 기능 개선 | Before/After 표 중심, A4 1장 이내 |
| 후속 연계 과업 | 배경 1문단 + 변경 표 + Scope만 |
